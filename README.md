# 🚗 カーアシスタント — 音声ナビゲーションデモ

**OpenAI Agents SDK（Voice Agents）**、Next.js、カスタムフェイクマップエンジンを使って構築したリアルタイム音声ナビゲーションアシスタントです。エージェントは近くの交通標識を自動的に警告し、交通に関する質問に音声で回答します。

---

## ✨ 機能一覧

### 基本機能
- **音声エージェント** — OpenAI Realtime API（WebRTC）で動作。プッシュトゥトークなしでリアルタイムに話し・聞きます。
- **フェイクマップエンジン** — 座標ベース（100×100グリッド）のSVGマップ。道路レイアウトは固定で、交通標識はランダム生成されます。
- **自動警告システム** — 車がマップに配置されると、エージェントがマップコンテキストツールを呼び出し、関連する警告を即座に案内します。
- **交通Q&A** — 「ここに駐車できますか？」「制限速度は？」など、現在のマップデータに基づいた正確な回答を音声で提供します。
- **会話ログ** — ユーザーとエージェントの音声がリアルタイムでテキスト化され、画面に表示されます。

### 独自機能 — 進行方向に基づく近接警告
エージェントは車の**進行方向の前方にある標識のみを警告**します。後方や並行する道路の標識は無視されます。

- **北/南方向**に走行中 → 同じ**縦方向の道路**上の標識のみハイライト
- **東/西方向**に走行中 → 同じ**横方向の道路**上の標識のみハイライト
- 標識は **現在地**（赤、12ユニット以内）と **前方**（黄、25ユニット以内）に分類されます

## 🤖 使用AIモデル

| 用途 | モデル | 説明 |
|---|---|---|
| **音声エージェント** | `gpt-4o-mini-realtime-preview` | リアルタイムの音声会話を担当。ユーザーの音声を処理し、ツールを呼び出して、WebRTC経由で低遅延の音声応答を生成します。 |
| **音声テキスト変換** | `gpt-4o-mini-transcribe` | ユーザーの音声入力をリアルタイムでテキストに変換し、画面上の会話ログに表示します。 |

### なぜ `gpt-4o-mini-realtime-preview` を使うのか？
- **音声to音声処理**: STT → LLM → TTSという個別パイプラインを介さず音声を直接処理するため、遅延が低い。
- **ツール呼び出し**: リアルタイムセッション内でfunctionツールをネイティブサポート。会話中にマップコンテキストの取得・駐車ルール確認・制限速度の取得が可能。

## 🚀 インストール & セットアップ

### 1. リポジトリをクローン

```bash
git clone <リポジトリURL>
cd car-assistant
```

### 2. 依存パッケージをインストール

```bash
npm install
# または
yarn install
```

### 3. 環境変数を設定

サンプルファイルをコピーしてAPIキーを設定します：

```bash
cp .env.local.example .env.local
```

`.env.local` をエディタで開き、プレースホルダーを書き換えます：

```env
OPENAI_API_KEY=sk-proj-your-key-here
```

### 4. 開発サーバーを起動

```bash
npm run dev
# または
yarn dev
```

ブラウザで [http://localhost:4000](http://localhost:4000) を開きます。

### 5. （任意）ngrokでマイクアクセスを有効化

ブラウザがマイクのアクセス権を付与するには **HTTPS** が必要です。リモートデバイスでテストしたり、公開HTTPSのURLが必要な場合は、[ngrok](https://ngrok.com/) を使ってローカルサーバーをトンネリングします：

```bash
ngrok http 4000
```

ngrokが `https://xxxx.ngrok.io` 形式の公開URLを発行します。このURLをブラウザで開くと、マイクへのアクセス権が正常に付与されます。

## 🎮 デモの流れ

1. **エージェント起動** — 緑色の「🎙️ エージェント起動」ボタンをクリック。マイクのアクセス許可が求められたら許可してください。
2. **ランダムゾーン** — 「ランダムゾーン」をクリックしてマップエリアを読み込みます（渋谷・都心部・スクールゾーン・高速道路入口・閑静な住宅街）。
3. **ランダム位置** — 「ランダム位置」をクリックして、マップ上にランダムな位置で車を配置します。
4. **警告を聞く** — エージェントが現在位置と進行方向に基づいて、近くの交通標識の警告を自動的に案内します。
5. **質問する** — 自然な言葉で話しかけます：
   - 「ここに駐車できますか？」
   - 「制限速度は何キロですか？」
   - 「前方に信号機はありますか？」

## 🗺 マップデータ

### 道路グリッド（固定 — 全ゾーン共通）

すべてのゾーンは、100×100の座標空間上の同じ **3×4道路グリッド** を使用します：

| 道路 | 方向 | 座標 | 名称 |
|---|---|---|---|
| h1 | 横方向 | y = 20 | 北通り |
| h2 | 横方向 | y = 55 | メイン通り |
| h3 | 横方向 | y = 88 | 南通り |
| v1 | 縦方向 | x = 20 | 一番街 |
| v2 | 縦方向 | x = 40 | 二番街 |
| v3 | 縦方向 | x = 60 | 三番街 |
| v4 | 縦方向 | x = 80 | 四番街 |

これにより **12の交差点**（例：`(20,20)`、`(40,55)`、`(80,88)`）と、標識を配置できる **7つの道路入口スロット** が生成されます。

---

### ゾーン一覧（全5種）

各ゾーンは同じ道路レイアウトを共有しますが、「ランダムゾーン」をクリックするたびに**異なる標識構成**がランダム生成されます。19の配置スロット（交差点12＋道路入口7）に、各ゾーンの優先標識タイプから60%の確率で標識が配置されます。

| ゾーン | 標識数 | 優先標識タイプ | 説明 |
|---|---|---|---|
| **渋谷** | 8 | 駐車禁止・信号機・横断歩道・一時停止 | 厳しい駐車規制のある繁華街 |
| **都心部** | 9 | 信号機・制限速度30・駐車禁止・徐行 | 交差点の多い市街地中心部 |
| **スクールゾーン** | 8 | 制限速度30・横断歩道・一時停止・駐車禁止 | 低速制限のある学校近くの住宅街 |
| **高速道路入口** | 7 | 制限速度80・制限速度60・進入禁止・徐行 | 高速道路の出入口エリア |
| **閑静な住宅街** | 6 | 制限速度30・横断歩道・徐行・一時停止 | のんびりした住宅地 |

## 🔑 エージェントツール

音声エージェントは5つのツールを使用します：

| ツール | 説明 |
|---|---|
| `get_current_context` | 車の位置・進行方向・近くの標識を返す |
| `check_parking` | 現在位置で駐車が許可されているか確認する |
| `get_speed_limit` | 最寄りの速度標識に基づいた制限速度を返す |
| `get_zone_info` | 現在のゾーンに関する一般情報を返す |
| `get_all_signs` | 現在のゾーン内のすべての標識を一覧表示する |
